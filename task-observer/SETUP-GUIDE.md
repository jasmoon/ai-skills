# Task Observer - Setup Guide

Choose your setup based on how you'll use this skill.

## Which Setup for You?

### Personal Projects Setup
**For:** Mobile + multiple local desktops (laptop, tablet, phone)
**Best for:** Cross-device work, capturing ideas anywhere
**Key feature:** Handoff doc mode syncs observations across devices

**Choose this if:**
- You work on the same projects from different devices
- You use Claude.ai Web on mobile
- You want to sync observations across devices using cloud storage (Dropbox, iCloud, etc.)
- You occasionally work from different locations

### Work Projects Setup
**For:** Local desktops only (no mobile access needed)
**Best for:** Persistent local storage, team projects
**Key feature:** Full filesystem access, persistent observation logs

**Choose this if:**
- You only work from your desktop(s)
- You want persistent local observation storage
- You prefer Claude Code as primary interface
- Your projects live in a shared local workspace

## Quick Comparison

| Feature | Personal | Work |
|---------|----------|------|
| Mobile access | ✅ Via Claude.ai Web | ❌ |
| Multiple devices | ✅ With handoff docs | ✅ Multiple desktops |
| Persistent logs | ✅ Cloud synced | ✅ Local filesystem |
| Setup complexity | Medium | Low |
| Primary interface | Claude.ai Web | Claude Code |
| Sync method | Manual handoff docs | Automatic (local) |

---

## After Setup

Both setups will:
- Automatically capture skill improvement opportunities
- Create observation logs during work sessions
- Help you build and improve reusable skills
- Work silently without interrupting your flow

The difference is mainly where observations get stored and how they're accessed across devices.
